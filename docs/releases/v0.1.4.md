# NameLens v0.1.4 Release Notes

Release date: 2026-01-25

## Highlights

v0.1.4 adds brand mark generation and thumbnail utilities, completing the
name-to-visual journey.

### Brand Mark Generation

New `namelens mark` command generates logo/brand mark concepts and images:

- AI-generated images via OpenAI GPT Image models
- Context-aware generation with `--description` and `--audience` flags
- Color control: `monochrome`, `brand`, `vibrant` palettes
- Transparent background support for compositing
- Multiple output formats: PNG, WebP, JPEG

### Thumbnail Utility

New `namelens image thumb` command creates agent-friendly thumbnails:

- Supports WebP, PNG, and JPEG input
- Configurable output size (64-1024px)
- Smart suffix handling (won't thumbnail existing thumbnails)
- Good for sharing, presentations, and AI agent workflows

### Dogfooding Success

We used namelens to create our own icon. The "network viewfinder" mark was
generated using `namelens mark`, validated through TinEye (0 matches across
80.9 billion images), and adopted as our official brand mark.

[Read the brand mark story →](../examples/namelens-brand-mark-story.md)

## Brand Mark Generation

Generate early-stage logo directions for finalist names:

```bash
# Basic generation
namelens mark "myproject" --out-dir ./marks --count 3

# Recommended: with context and color
namelens mark "myproject" --out-dir ./marks \
  --description "Static analysis tool for Go" \
  --audience "developers" \
  --color brand \
  --background transparent \
  --format png \
  --quality high
```

Output:

```
./marks/
├── myproject_brand-mark_01.png
├── myproject_brand-mark_02.png
└── myproject_brand-mark_03.png
```

### Color Modes

Control the output palette with `--color`:

| Mode | Description |
|------|-------------|
| `brand` | Modern tech palette (teal, navy, coral accents) |
| `monochrome` | Black, white, and greys |
| `vibrant` | Bold, saturated startup colors |

```bash
# Tech-forward brand colors
namelens mark "myproject" --out-dir ./marks --color brand
```

### Context Flags

Provide context for better results:

```bash
namelens mark "myproject" --out-dir ./marks \
  --description "What your project does" \
  --audience "Who it's for"
```

Without context, the model infers from the name alone. With context, marks
reflect what you're building.

## Thumbnail Generation

Create smaller versions for sharing:

```bash
# Generate JPEG thumbnails
namelens image thumb --in-dir ./marks --max-size 256 --format jpeg

# Custom output location
namelens image thumb --in-dir ./marks --out-dir ./thumbs
```

Output:

```
./marks/
├── myproject_brand-mark_01.png
├── myproject_brand-mark_01.thumbnail.jpg
└── ...
```

Thumbnails are useful for:

- Slack/Discord/email sharing
- AI agent context (smaller token cost)
- Quick stakeholder reviews

## Provider Configuration

Route text and image generation separately for best results:

```bash
# .env configuration
NAMELENS_AILINK_ROUTING_BRAND_MARK=namelens-openai
NAMELENS_AILINK_ROUTING_BRAND_MARK_IMAGE=namelens-openai-image

# Dedicated image provider
NAMELENS_AILINK_PROVIDERS_NAMELENS_OPENAI_IMAGE_ENABLED=true
NAMELENS_AILINK_PROVIDERS_NAMELENS_OPENAI_IMAGE_AI_PROVIDER=openai
NAMELENS_AILINK_PROVIDERS_NAMELENS_OPENAI_IMAGE_MODELS_IMAGE=gpt-image-1.5
```

### Provider Recommendations

| Provider | Status | Notes |
|----------|--------|-------|
| OpenAI GPT Image | Recommended | Best instruction following |
| xAI grok-2-image | Experimental | No size/aspect control |

See [known-issues.md](../known-issues.md) for xAI limitations.

## Workflow Integration

Recommended workflow with v0.1.4:

```
generate → compare → review → mark → thumb
```

1. **Generate** candidates with `namelens generate`
2. **Compare** finalists with `namelens compare`
3. **Review** top picks with `namelens review`
4. **Mark** chosen name with `namelens mark --color brand`
5. **Thumb** for sharing with `namelens image thumb`

## Technical Details

### Mark Command Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--out-dir` | (required) | Output directory |
| `--count` | 3 | Number of marks to generate |
| `--description` | | Product description for context |
| `--audience` | | Target audience for context |
| `--color` | | Color mode: monochrome, brand, vibrant |
| `--format` | png | Output format: png, jpeg, webp |
| `--background` | auto | Background: auto, transparent, opaque |
| `--quality` | auto | Quality: auto, low, medium, high |
| `--size` | 1024x1024 | Image dimensions |

### Thumb Command Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--in-dir` | (required) | Input directory |
| `--out-dir` | (in-dir) | Output directory |
| `--max-size` | 256 | Max dimension (64-1024) |
| `--format` | jpeg | Output format: jpeg, png |
| `--suffix` | thumbnail | Filename suffix |

## Upgrade Notes

No breaking changes. All v0.1.3 configurations and caches remain valid.

New environment variables for image generation are optional - the tool works
without them but will require configuration for `namelens mark`.

## Known Limitations

- xAI image generation is experimental (no size/aspect control)
- Generated marks are directional concepts, not final artwork
- AI-generated images cannot be copyrighted (per current legal precedent)

See [USAGE-NOTICE.md](../USAGE-NOTICE.md) for important disclaimers on
AI-generated content and trademark considerations.
