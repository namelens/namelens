# NameLens v0.1.0 Release Notes

Release date: 2026-01-04

## Highlights

Initial release of NameLens - a CLI tool for checking name availability across
domains, package registries, and social handles.

### Core Features

- **Domain availability** via RDAP bootstrap with caching and rate limits
- **WHOIS/DNS fallback** for TLDs without RDAP support
- **Registry checks** for npm, PyPI, and GitHub
- **Expert analysis** via AILink prompt library (xAI/Grok)
- **Batch checks** with table/json/markdown output formats

### Name Generation

- **Generate command** for AI-powered naming ideation
- **Phonetics analysis** for pronunciation and typeability scoring
- **Suitability analysis** for cultural appropriateness screening

### Developer Experience

- Built-in profiles: `startup`, `developer`, `website`, `minimal`, `web3`
- Tri-state availability: available/taken/unknown
- Diagnostics commands: `envinfo`, `doctor`, `doctor ailink`,
  `doctor ailink connectivity`
- XDG-compliant configuration paths

## Configuration

- Default store: `$XDG_DATA_HOME/namelens/namelens.db` (local libsql)
- Expert provider: AILink `ai_provider: xai` (Grok API)
- WHOIS fallback enabled by default

## Known Limitations

- WHOIS/DNS fallbacks rely on pattern matching; results may be ambiguous
- RDAP uses the first server per TLD; no multi-server retry yet
- Some TLDs (.dev, .app) return `unknown` until RDAP support is added
