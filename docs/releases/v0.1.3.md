# NameLens v0.1.3 Release Notes

Release date: 2026-01-14

## Highlights

v0.1.3 adds candidate comparison tooling and Rust ecosystem support.

### Compare Command

New `namelens compare` command for side-by-side candidate screening before
running expensive brand analysis:

- Compare 2-10 names with availability, risk, phonetics, and suitability scores
- Quick mode (`--mode=quick`) for fast availability-only screening
- Full mode includes AI-powered phonetics and cultural suitability analysis
- Export to table, JSON, or markdown formats

### Cargo/crates.io Checker

Rust crate availability now included in registry checks:

- New `cargo` registry option alongside npm and pypi
- Developer profile updated to include cargo by default
- Shows crate metadata (version, description, repository) for taken names

### Review Quick Mode

New `--mode=quick` flag for the review command enables fast screening workflows
without full brand analysis overhead.

### Reliability Improvements

- .app and .dev domains now route via Google RDAP for reliable availability
  checks
- SQLite store tolerates lock during WAL mode setup

## Compare Command

Screen 2-10 candidates before running expensive brand analysis:

```bash
# Quick availability screening (~10 seconds)
namelens compare fulgate agentanvil toolcrux --mode=quick
```

Output:

```
╭────────────┬──────────────┬────────╮
│ NAME       │ AVAILABILITY │ LENGTH │
├────────────┼──────────────┼────────┤
│ fulgate    │ 7/7          │      7 │
│ agentanvil │ 7/7          │     10 │
│ toolcrux   │ 7/7          │      8 │
╰────────────┴──────────────┴────────╯
```

Full analysis with phonetics and suitability:

```bash
namelens compare fulgate toolcrux
```

Output:

```

│ NAME     │ AVAILABILITY │ RISK │ PHONETICS │ SUITABILITY │ LENGTH │
```

Export for stakeholders:

```bash
namelens compare alpha beta gamma --output-format=markdown --out comparison.md
```

## Cargo/crates.io Support

Check Rust crate availability:

```bash
# Check specific registry
namelens check mycrate --registries=cargo

# Developer profile now includes cargo
namelens check myproject --profile=developer

# JSON output shows crate metadata
namelens check serde --registries=cargo --output-format=json
```

Example output for a taken crate:

```json
{
  "name": "serde",
  "check_type": "cargo",
  "available": "taken",
  "extra_data": {
    "name": "serde",
    "version": "1.0.215",
    "description": "A generic serialization/deserialization framework",
    "repository": "https://github.com/serde-rs/serde"
  }
}
```

## Review Quick Mode

Fast screening without full brand analysis:

```bash
namelens review myproject --mode=quick
```

Quick mode skips expensive brand-plan analysis, returning only availability and
basic risk assessment.

## Workflow Integration

Recommended naming workflow with v0.1.3:

```
generate → compare --mode=quick → compare (full) → review
```

1. **Generate** candidates with `namelens generate`
2. **Quick screen** with `namelens compare --mode=quick` to filter by
   availability
3. **Full compare** with `namelens compare` to add phonetics/suitability scores
4. **Deep review** with `namelens review` on 1-2 finalists

## Technical Details

### Compare Command Flags

| Flag              | Default   | Description                                |
| ----------------- | --------- | ------------------------------------------ |
| `--mode`          | (full)    | `quick` for availability only              |
| `--profile`       | `startup` | Availability profile                       |
| `--output-format` | `table`   | Output format: table, json, markdown       |
| `--out`           | stdout    | Write output to file                       |
| `--no-cache`      | false     | Skip cache, force fresh lookups            |

### Risk Level Derivation

Compare derives risk level from availability results without AI calls:

- **high**: .com is taken
- **medium**: some assets taken but .com available
- **low**: all or most assets available

## Upgrade Notes

No breaking changes. All v0.1.2 configurations and caches remain valid.

## Known Limitations

- Compare supports 2-20 names per invocation
- Phonetics/suitability may show `-` if AI analysis times out (partial results
  still useful)
